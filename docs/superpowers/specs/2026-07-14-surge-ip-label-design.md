# Surge 节点 IP 标签加工器设计

## 目标

在不更改上游 Sub-Store 聚合配置或机场订阅的前提下，生成一份可供 Surge `policy-path` 使用的私有加工订阅。Surge 的原生节点选择和延迟测试列表将显示加工后的节点名称，便于按出口 IP 情报判断节点。

目标名称格式如下：

```text
IPLC 香港 01 [203.0.113.42] 🟢92 | 原生IP | 住宅 | 人类偏多
```

名称保留原节点名作为前缀，避免影响现有 `policy-regex-filter` 中的地区、IPLC 和供应商关键字。

## 范围与边界

- 上游为用户现有的 Sub-Store Surge 输出 URL，仅以只读方式拉取。
- 不改变 Sub-Store 的集合、机场订阅、规则或原始 URL。
- 输出保留每个代理的协议、服务器、端口和全部参数；仅修改等号左边的节点显示名。
- 不将节点凭据、上游 URL 或访问令牌写入 Git 仓库、GitHub Pages 或日志。
- GitHub Pages 仅托管公开状态页和使用说明；包含代理定义的加工订阅由受令牌保护的私有接口返回。
- 不尝试用 Surge 模块改变既有 `[Proxy Group]`；模块不具备修改这些组或其远程策略名称的能力。

## 架构

1. Surge 定时脚本从用户在模块参数中填写的上游 Sub-Store URL 读取 Surge 格式节点定义。
2. 脚本解析每个代理行，并用 Surge `$httpClient` 的 `policy-descriptor` 逐个建立临时出站请求，获得实际出口完整 IP。
3. 脚本以单并发安全队列查询 Net.Coffee，读取评分、原生性、住宅/机房和人机流量信号，并保存 24 小时缓存。
4. 脚本格式化显示名，保留原代理行右侧的连接参数不变，并将加工后的外部策略文件上传到私有订阅接口。
5. 私有订阅接口以访问令牌验证请求后提供最新文件；公开状态页只展示时间、成功数和失败数等无凭据指标。
6. Surge 配置补丁把所有读取上游的 `policy-path` 迁移到私有加工订阅，并使用 `update-interval=3600` 重新拉取外部策略。

## 字段映射与降级

| 展示字段 | 数据来源 | 降级行为 |
| --- | --- | --- |
| 完整 IP | 节点实际出站的 IP 回显结果 | `IP:未知` |
| 评分 | Net.Coffee `trust_score` | `评分:未知` |
| 原生性 | Net.Coffee 返回的原生/非原生信号 | `原生性未知` |
| 网络属性 | 住宅、机房、移动网络等信号 | `类型未知` |
| 人机流量 | 人类偏多、爬虫或其他实际返回的分类 | `人机未知` |

评分颜色只使用文本 emoji：80–100 为 `🟢`，50–79 为 `🟡`，0–49 为 `🔴`，未知不加颜色符号。不会虚构 `Bot:12%`；只有数据源明确提供数值时才显示百分比。

## 失败与安全处理

- IP 情报查询使用单并发队列；同一出口 IP 在一轮中只查询一次。
- 每个查询间隔至少 3 秒，并加入随机抖动。
- 查询缓存有效期为 24 小时。节点出口 IP 未变且缓存未过期时，不再请求 Net.Coffee。
- 节点连接或 IP 回显失败时，输出节点仍保留在订阅中，名称标记为 `检测失败`，避免单个故障节点导致订阅缺失。
- Net.Coffee 查询失败时使用已缓存的、未过期的结果；无缓存时标记相应字段为未知。
- 收到 `429`、`403`、任何 `5xx` 或连续超时后，立即停止当前轮次并至少等待 24 小时再恢复查询。
- 所有日志仅记录节点序号、哈希化名称和错误类别，不记录完整代理行、凭据、订阅 URL 或令牌。
- 加工结果需由私有订阅接口提供。公开 GitHub Pages 绝不存储或引用完整节点定义。

## 验证标准

1. 用含 HTTP、SOCKS5、SS、VMess、Trojan 等典型节点的样本验证：输出行除名称外字节等价。
2. 将生成文件作为 `policy-path` 加载，Surge 配置检查通过且原生选择列表显示加工名称。
3. 现有地区/IPLC/供应商正则仍能筛选到同一批节点。
4. 节点名称中可见完整出口 IP、评分、原生性、网络属性和真实可用的人机流量标签。
5. 公开站点与 Git 历史中不存在代理凭据、上游订阅 URL 或令牌。
